# 云达诺 V1.0.1 版本更新说明

## 版本信息
- **版本号**: V1.0.1
- **更新日期**: 2025年12月5日
- **项目名称**: 云达诺 FM33LG0XX 测试系统

## 主要更新内容

### 1. 工位检测功能优化
- **扩展工位识别范围**: 将原有的5个工位扩展到10个工位(0x00-0x09)
- **新增GPIO引脚**: 增加了`GPIOE_PIN_5`作为工位识别的第四个输入引脚
- **改进识别逻辑**: 通过4个GPIO引脚组合,支持更精确的工位识别
- **文件修改**: `Src/Test_List.c` - `gongwei_jiance()`函数

### 2. 测试流程优化
- **新增阀门测试流程**: 在主动上报之前增加独立的阀门测试环节
  - 新增测试状态: `w_famenceshi`(阀门测试状态)
  - 实现实时阀门动作检测功能
  - 优化阀门开关检测逻辑
- **简化测试流程**: 移除了独立的阀门开关测试步骤,集成到统一的测试流程中
- **调整超时时间**: 将某些测试步骤的超时时间从20秒调整为25秒

### 3. 阀门检测功能增强
- **新增实时检测函数**: `Test_loop_func()`
  - 实时监测阀门1和阀门2的电压状态
  - 自动进行到位检测和状态切换
  - 优化阀门开关逻辑
- **阀门状态记录**:
  - `FM1`: 阀门1状态(0x01-正常, 0x02-异常, 0x03-完成测试)
  - `FM2`: 阀门2状态(0x01-正常, 0x02-异常, 0x03-完成测试)
  - `FM_daowei`: 阀门到位检测标志

### 4. 计量方式调整
- **固定计量方式**: 将计量方式固定为"有磁"模式
- **注释旧逻辑**: 保留原有通过GPIO检测计量方式的代码(已注释)
- **统一传感器控制**: 在固定模式下统一设置传感器引脚状态

### 5. 压力检测功能更新
- **字段重命名**: 将`lanya_jiance`(蓝牙检测)重命名为`yali_jiance`(压力检测)
- **检测逻辑优化**: 
  - 结合压力检查标志(`L_yali_CHK`)
  - 结合外部中断标志(`EXTIFlag`)
  - 双重确认机制确保检测准确性
- **协议支持**: 在PC协议中增加压力检测结果的上报

### 6. 测试查询流程简化
- **移除冗余检测**: 在第二次查询结果时,移除了以下冗余检测:
  - 磁干扰检测相关代码
  - 无磁计量重复检测
  - 阀门到位重复检测
- **保留核心检测**: 仅保留必要的Flash、EEPROM等核心功能检测

### 7. 主循环整合
- **新增调用**: 在`main.c`主循环中新增`Test_loop_func()`函数调用
- **实现周期性检测**: 实现阀门状态的周期性实时监测

### 8. 代码结构优化
- 移除了`w_famen_guan`(阀门关)和`w_famen_kai`(阀门开)独立状态
- 整合阀门测试到统一的`w_famenceshi`状态
- 优化了测试流程的状态机转换逻辑

## 技术改进

### 可靠性提升
- 增强了阀门检测的可靠性,采用实时电压监测
- 优化了工位识别的准确性,支持更多工位配置
- 改进了超时机制,提高系统稳定性

### 可维护性提升
- 简化了测试流程状态机
- 移除冗余检测代码
- 统一了命名规范(如lanya→yali)

### 功能性增强
- 支持10个工位的精确识别
- 新增独立的阀门测试环节
- 实现阀门状态的实时监控

## 使用说明

1. 本版本在原有基础上进行了重大功能优化
2. 测试流程更加完善,增加了阀门专项测试
3. 工位识别能力增强,支持更大规模的生产测试
4. 压力检测功能取代了原有的蓝牙检测功能

## 兼容性说明

- 硬件要求: 需要确保`GPIOE_PIN_5`已正确连接用于工位识别
- 阀门电压检测: 需要确保阀门电压检测电路正常工作(阈值1000mV)
- 测试夹具: 适配10工位测试夹具
